Consent Test with Inmobi

A Consent Manager Platform (CMP), Inmobi, was implemented and configured on this page.
Before configuring GTM, the tags firing on this page will be ignoring the consent given. The task is blocking tags according to consent, and implementing consent mode.
There are also 3 buttons you can use to test datalayer events.

Block Google and Media Tags in GTM
1. Import the Inmobi GTM container (2.2.8 folder), with its tags and variables that enable sending consent information to the dataLayer.
2. Block all analytics and media tags to fire according to the consent given.

Implement Consent Mode
1. Open the InMobi Dashboard (use the analytics email to login). Find and enable the consent mode options.
2. Implement the basic implementation of consent mode.
3. Implement the advanced implementation of consent mode with URL passthrough as granted and ads data redaction as denied.

To reset cookies, open devtools and go to Application > Storage and click on 'Clear site data' (don't include third party cookies).
To check if consent mode is implemented on the page, paste the following code in the console: google_tag_data.ics.entries
You can also find more guides and information on the 2.2.8 folder.
